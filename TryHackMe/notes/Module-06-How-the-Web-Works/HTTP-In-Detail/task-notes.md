## Overview

This room introduced the HyperText Transfer Protocol (HTTP) and its secure counterpart (HTTPS). It covered the request and response lifecycle, URL structure, HTTP methods, status codes, headers, cookies, and practical request creation.

---

## Objectives

- Understand HTTP and HTTPS
- Learn URL structure
- Understand HTTP requests and responses
- Learn HTTP methods
- Interpret HTTP status codes
- Understand request and response headers
- Learn how cookies work
- Practice constructing HTTP requests

---

## Key Concepts

### HTTP vs HTTPS

- HTTP is used for web communication.
- HTTPS encrypts communication using TLS.

### URL Components

- Scheme
- Host
- Port
- Path
- Query String
- Fragment

### HTTP Methods

| Method | Purpose |
|---------|---------|
| GET | Retrieve data |
| POST | Create new resources |
| PUT | Update existing resources |
| DELETE | Remove resources |

### Status Codes

- 1xx Informational
- 2xx Success
- 3xx Redirection
- 4xx Client Errors
- 5xx Server Errors

Common:

- 200 OK
- 201 Created
- 301 Moved Permanently
- 302 Found
- 400 Bad Request
- 401 Unauthorized
- 403 Forbidden
- 404 Not Found
- 500 Internal Server Error
- 503 Service Unavailable

### Headers

#### Request

- Host
- User-Agent
- Content-Length
- Accept-Encoding
- Cookie

#### Response

- Set-Cookie
- Cache-Control
- Content-Type
- Content-Encoding

### Cookies

- Store session information
- Maintain authentication
- Remember user preferences

### Practical Skills

- Construct GET requests
- Construct POST requests
- Construct PUT requests
- Construct DELETE requests
- Configure query parameters
- Configure body parameters
