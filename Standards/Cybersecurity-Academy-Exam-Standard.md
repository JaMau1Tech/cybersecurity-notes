# Root & Repository Cybersecurity Academy Exam Standard

**Version:** 2.0

**Status:** 🔒 LOCKED

**Last Updated:** 2026-07-12

This document defines the official standard used when creating, administering, and grading every written assessment within the Root & Repository Cybersecurity Academy.

This standard applies to:

- Pre-Security Midterm
- Module 05
- Module 06
- Module 07
- Pre-Security Final

This document defines **how exams are built**, while the **Cybersecurity Academy Workflow** defines **when assessments are completed**.

No standards may be modified without explicit user approval.

---

# Core Principles

Every written assessment should:

- Reinforce long-term knowledge retention.
- Measure understanding rather than memorization.
- Evaluate technical reasoning and problem solving.
- Simulate a professional certification exam.
- Prepare the learner for real-world IT and cybersecurity situations.

# Question Design Philosophy

Every assessment should progress through four levels.

## Level 1 — Recall

Can the learner remember the information?

---

## Level 2 — Understanding

Can the learner explain the concept?

---

## Level 3 — Application

Can the learner apply the concept?

---

## Level 4 — Troubleshooting

Can the learner diagnose and solve a problem?

---

# Question Quality Standards

Every question must:

- Be technically accurate.
- Have one clearly correct answer unless it is a Multiple Select question.
- Avoid trick wording.
- Avoid ambiguity.
- Reflect realistic IT and cybersecurity situations.
- Focus on reasoning and practical application.

---

# Grading Standards

## Objective Questions

The following sections are graded as Correct or Incorrect:

- Multiple Choice
- True / False
- Fill in the Blank
- Matching

---

## Multiple Select

Multiple Select questions use partial credit.

Credit is awarded based on:

- Correct selections
- Incorrect selections
- Completeness of the answer

---

## Subjective Questions

The following sections are graded using a rubric:

- Short Answer
- Scenario-Based
- Troubleshooting

Rubrics evaluate:

- Technical accuracy
- Completeness
- Logical reasoning
- Communication

Partial credit is permitted.

---

# Exam Administration

During an active assessment the assistant must:

- Never reveal answers.
- Never provide hints.
- Never explain questions.
- Never indicate whether an answer is correct.
- Never assist with solving questions.

These restrictions remain in place until the learner submits the completed assessment.

---

# Assessment Completion

Once the assessment has been submitted:

1. Grade the assessment.
2. Generate `results.md`.
3. Continue the Cybersecurity Academy Workflow.
4. Provide feedback based on demonstrated knowledge.
5. Generate a targeted review guide covering only missed concepts.

---

# Assistant Responsibilities

The assistant must:

- Follow this standard for every Academy written assessment.
- Follow the Cybersecurity Academy Workflow.
- Maintain professional certification standards.
- Grade consistently.
- Apply grading rubrics fairly.
- Focus on understanding and practical application rather than memorization.
- Never modify this standard without explicit user approval.
