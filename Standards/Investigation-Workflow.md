# Investigation Workflow Standard

**Version:** 1.0

**Status:** 🔒 LOCKED

**Last Updated:** 2026-07-11

This document defines the official workflow for every investigation completed as part of the Root & Repository project.

No steps may be skipped, reordered, added, or removed without explicit user approval.

---

# Core Principles

- Every investigation follows the same documented workflow.
- Investigations are designed to reinforce technical skills through hands-on practice.
- Commands should be understood, not just executed.
- Documentation should be concise, accurate, and reproducible.
- Every completed investigation is committed to Git.

---

# Repository Structure

The Investigations repository is organized by technology.

Example:

Investigations/

├── Linux/
├── Networking/
└── Windows/

Each investigation is stored inside the appropriate technology folder.

Example:

Windows/

└── Investigation-01-System-Recon/
    ├── screenshots/
    └── investigation-notes.md

No additional folders or documentation files are created unless explicitly approved.

---

# Investigation Workflow

Every investigation follows this exact structure.

## 1. Objective

Briefly describe the purpose of the investigation.

---

## 2. Notebook Notes

Create concise handwritten notes before beginning the hands-on work.

Include:

- Important commands
- Key concepts
- Definitions
- Tools used

---

## 3. Lab Steps

Break the investigation into tasks.

Each task includes:

- Task title
- Commands
- What to record or observe

Example:

Task 1

Commands

Record

---

## 4. Hands-On Investigation

Complete each task.

Run the commands.

Observe the output.

Record important findings.

---

## 5. Screenshot Workflow

After completing a significant task:

📸 Stop.

Prompt the user to take a screenshot.

Provide the exact filename.

Wait for confirmation.

Continue.

---

## 6. Screenshot Checklist

Generate the complete screenshot checklist.

Include every screenshot captured during the investigation.

---

## 7. Lessons Learned

Summarize:

- Technical concepts learned
- Commands practiced
- Skills reinforced
- Practical takeaways

---

# Investigation Documentation

Each investigation contains a single:

investigation-notes.md

This document includes:

- Objective
- Notebook Notes
- Lab Steps
- Screenshot Checklist
- Lessons Learned

No additional documentation files are created unless explicitly approved.

---

# Screenshot Standard

Screenshots should capture:

- Successful command execution
- Important output
- Significant investigation milestones
- Final results

Every screenshot uses the approved Screenshot Standard.

---

# Git Workflow

Every completed investigation concludes with:

1. Repository review
2. Git status
3. Git add
4. Git commit
5. Git push

The assistant provides the exact Git commands.

---

# Quality Assurance Checklist

Before an investigation is considered complete:

- [ ] Objective documented
- [ ] Notebook notes completed
- [ ] Lab tasks completed
- [ ] Commands executed successfully
- [ ] Screenshots captured
- [ ] Screenshot checklist completed
- [ ] Lessons Learned completed
- [ ] investigation-notes.md reviewed
- [ ] Repository reviewed
- [ ] Git commands generated

Only after every box is complete is the investigation finished.

---

# Assistant Responsibilities

The assistant must:

- Follow this workflow exactly.
- Never invent commands or findings.
- Prompt for screenshots immediately after significant tasks.
- Wait for confirmation before continuing.
- Generate documentation only after the investigation is complete.
- Never modify the investigation workflow without explicit user approval.
