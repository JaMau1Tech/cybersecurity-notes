# Workflow Order

Career development follows this exact order.

## Phase 1

Career README

---

## Phase 2

Resume

1. Resume-Master.md
2. Resume-Master.docx
3. Resume.pdf
4. Resume-IT.md
5. Resume-IT.docx
6. Resume-Cybersecurity.md
7. Resume-Cybersecurity.docx

The Resume Master is the single source of truth.

---

## Phase 3

LinkedIn

Complete every LinkedIn document before updating the live profile.

---

## Phase 4

Cover Letters

Create:

- IT Cover Letter
- Cybersecurity Cover Letter
- Master Template

---

## Phase 5

Interview Preparation

Complete every Interview-Prep document before beginning active interviewing.

---

## Phase 6

Job Tracker

Maintain:

- Applications
- Interviews
- Networking
- Follow-Ups
- Company Wishlist
- Weekly Goals
- Monthly Reviews

---

# Documentation Rules

Every document should:

- Use Markdown
- Follow consistent formatting
- Be reviewed before committing
- Reflect the current Resume Master

---

# Git Workflow

Every major Career milestone concludes with:

1. Repository review
2. Git status
3. Git add
4. Git commit
5. Git push

The assistant provides the exact Git commands.

---

# Quality Assurance Checklist

Before considering the Career repository complete:

- [ ] README completed
- [ ] Resume workflow completed
- [ ] LinkedIn workflow completed
- [ ] Cover Letter workflow completed
- [ ] Interview Preparation completed
- [ ] Job Tracker completed
- [ ] Repository reviewed
- [ ] Git commands generated

# Assistant Responsibilities

The assistant must:

- Follow this workflow exactly.
- Never change the workflow without explicit user approval.
- Maintain Resume-Master.md as the source of truth.
- Ensure all derivative resumes remain synchronized.
- Maintain consistent documentation throughout the Career repository.
