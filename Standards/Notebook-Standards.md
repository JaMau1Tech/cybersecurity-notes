# Notebook Standards

**Version:** 1.0

**Status:** 🔒 LOCKED

**Last Updated:** 2026-07-11

This document defines the official handwritten notebook standard used throughout the Root & Repository project.

This template is used for TryHackMe rooms, technical learning, and other educational activities unless a different workflow explicitly defines its own notebook format.

No changes may be made without explicit user approval.

---

# Core Principles

- Notes are written by hand to reinforce learning.
- Notes are concise and easy to copy into a notebook.
- Focus on understanding concepts rather than copying paragraphs.
- Use bullets instead of long paragraphs whenever possible.
- Include only information that supports long-term retention.

---

# Required Section Order

Every notebook entry must use this exact order.

---

# TOPIC

Title of the room or lesson.

---

# MODULE

The learning module the room belongs to.

---

# TASK

Current task name or number.

For room-complete notes:

Room name.

---

# DATE

Date completed.

---

# STATUS

- In Progress
- Complete

---

# OVERVIEW

Brief summary of what was learned.

Use concise bullet points.

---

# LEARNING OBJECTIVES

List the primary learning goals.

Use bullets.

---

# KEY TERMS

List important terminology introduced.

Examples:

- DNS
- DHCP
- Router
- Switch
- Hashing

---

# IMPORTANT CONCEPTS

Explain the major concepts using concise bullets.

Examples:

- What it is
- Why it matters
- Key characteristics

Do not write long paragraphs.

---

# LAB / HANDS-ON

Document practical activities completed.

Include:

- Commands used
- Tools used
- Actions performed
- Results

If no lab exists:

Write:

No hands-on activity completed.

---

# SCREENSHOT CHECKLIST

List every screenshot captured during the room.

Include:

- Task screenshots
- Lab screenshots
- Room completion screenshot

Use the approved Screenshot Standards.

---

# WHY THIS MATTERS IN CYBERSECURITY

Brief explanation connecting the lesson to real-world IT or cybersecurity.

---

# QUICK REVIEW

Short bullet-point recap of the most important takeaways.

Keep this section brief.

---

# PERSONAL REFLECTION

Write a short reflection including:

- What I learned
- What challenged me
- What I want to study next
- How this supports my long-term career goals

---

# Writing Standards

Use:

- Bullet points
- Short phrases
- Simple language
- Clear headings

Avoid:

- Large paragraphs
- Copying the room verbatim
- Unnecessary repetition

---

# Assistant Responsibilities

The assistant must:

- Generate notebook notes before completing the task.
- Use this exact section order.
- Keep notes concise for handwriting.
- Keep lab documentation accurate.
- Include every required section.
- Never invent lab activities.
- Never omit completed labs.
- Never change the notebook structure without user approval.
