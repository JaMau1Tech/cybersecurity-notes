# Root & Repository Cybersecurity Academy Grading Standard

**Version:** 1.0

**Status:** 🔒 LOCKED

**Last Updated:** 2026-07-12

This document defines the official grading methodology for every written and practical assessment within the Root & Repository Cybersecurity Academy.

This standard ensures every assessment is graded consistently, fairly, and professionally.

This document works alongside:

- Cybersecurity-Academy-Workflow.md
- Cybersecurity-Academy-Exam-Standard.md

No grading standards may be modified without explicit user approval.

---

# Core Principles

Every assessment should:

- Reward understanding over memorization.
- Reward logical reasoning.
- Reward practical application.
- Provide constructive feedback.
- Identify strengths and weaknesses.
- Produce repeatable and consistent grading.

---

# Passing Score

Minimum Passing Score:

**80%**

---

# Written Assessment Grading

## Objective Questions

The following sections are graded as Correct or Incorrect.

- Multiple Choice
- True / False
- Fill in the Blank
- Matching

Each question receives full credit or no credit.

---

## Multiple Select

Multiple Select questions may receive partial credit.

Points are awarded based on:

- Correct selections
- Incorrect selections
- Overall completeness

Selecting every correct option without selecting incorrect options receives full credit.

---

## Subjective Questions

The following sections are graded using rubrics.

- Short Answer
- Scenario-Based
- Troubleshooting

Partial credit is permitted.

---

# Short Answer Rubric

Each answer is evaluated using four categories.

| Category | Points |
|----------|-------:|
| Technical Accuracy | 3 |
| Completeness | 3 |
| Understanding | 2 |
| Communication | 2 |

Maximum:

**10 Points**

---

# Scenario-Based Rubric

Each answer is evaluated using four categories.

| Category | Points |
|----------|-------:|
| Problem Identification | 3 |
| Technical Reasoning | 3 |
| Recommended Solution | 2 |
| Explanation | 2 |

Maximum:

**10 Points**

---

# Troubleshooting Rubric

Each answer is evaluated using five categories.

| Category | Points |
|----------|-------:|
| Information Gathering | 2 |
| Logical Investigation | 2 |
| Technical Accuracy | 2 |
| Recommended Resolution | 2 |
| Communication | 2 |

Maximum:

**10 Points**

# results.md Standard

Every assessment generates a results.md containing:

- Assessment name
- Assessment version
- Overall score
- Percentage
- Pass / Fail
- Section-by-section performance
- Strengths
- Weaknesses
- Areas for improvement
- Readiness assessment

---

# reflection.md Standard

Reflection documents include:

- Personal strengths
- Areas requiring improvement
- Lessons learned
- Confidence level
- Study goals
- Next steps

---

# review-guide.md Standard

Review guides are generated using only missed concepts.

Do not include topics already demonstrated as mastered.

---

# answer-key.md Standard

Answer keys include:

- Correct answer
- Explanation
- Supporting concepts
- Reasoning

---

# Practical Assessment Grading

Practical assessments evaluate:

- Technical accuracy
- Methodology
- Troubleshooting
- Documentation
- Communication
- Professionalism

Partial credit is permitted.

---

# Mock Interview Evaluation

Mock interviews assess:

- Technical knowledge
- Communication
- Confidence
- Problem solving
- Professionalism

Interview feedback should include:

- Strengths
- Improvement areas
- Recommended study topics

---

# mastery-report.md Standard

Every mastery report summarizes:

- Overall assessment performance
- Technical strengths
- Technical weaknesses
- Demonstrated skills
- Knowledge gaps
- Recommended next steps
- Readiness to progress

---

# Assistant Responsibilities

The assistant must:

- Grade consistently using this standard.
- Apply rubrics fairly.
- Provide constructive feedback.
- Reward logical reasoning.
- Never inflate scores.
- Never penalize minor wording differences when technical understanding is demonstrated.
- Follow the Cybersecurity Academy Workflow and Cybersecurity Academy Exam Standard.
